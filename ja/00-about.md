# このサービスとは

まず prperf が**何ものか**をはっきりさせます。

## ひとことで

**prperf は、コードの変更で性能が悪化していないかを PR ごとに自動チェックして
PR に知らせる GitHub App です。** 測定はあなたの CI の中で OSS の
[rperf](https://github.com/ko1/rperf) が行い、prperf 自身は、**マージ先ブランチ
(ふつう main)の最新の計測**と **この PR の計測**を比べて結果を通知するだけの
薄い App です。

この「マージ先ブランチ側の基準」を **base**、PR 側を **head** と呼びます
(GitHub の PR 用語と同じ)。本マニュアルでは以降この呼び方を使います。

> テストカバレッジを CI で追う **Codecov** を知っていれば、その「性能版」と
> 思ってください。

rperf は**時間(CPU)ベースのサンプリングプロファイラ**で、本体は「どこで時間を
使ったか」を表す**フレームグラフ**です。prperf はそのプロファイルから
**実行時間・GC・アロケーション**を取り出し、base/head で比較します。PR を作ると、
Check Run にこんな**要約**が出ます:

> 2,001ms → 2,140ms (+7%) · alloc 48,741 → 59,950 (+23%) · GC 4 → 7

「このコミットで性能がどう変わったか」を、**マージ前の PR の時点で**気づける
ようにするのが prperf です。

## 何をするか / しないか

**する:**

- PR ごとに base→head の性能差(アロケーション・GC・時間)を Check Run に表示
- 閾値を超えたときだけ PR にコメント(sticky、通知は静か)
- フレームグラフの diff で「どのメソッドが重くなったか」を可視化

**しないこと:**

- **本番監視ではありません。** Datadog / Grafana の代替ではなく、補完です
  (本番の継続監視はそちら、PR 時点の回帰検知が prperf)。
- **CI を落としません。** 判定はあくまで参考表示。Check の conclusion は常に
  success です。
- **あなたのコードをサービス側で実行しません。** 測定は**あなたの CI の中**で
  行われ、prperf はその結果(プロファイル)を受け取って比較するだけ。これが
  「薄い App」として成立する理由で、セキュリティとコストの両面で軽い。

## 全体像

```
あなたの CI (GitHub Actions)
  └─ prperf-action
       ├─ rperf でベンチを N 回計測
       └─ プロファイル(.json.gz)を prperf サーバーへアップロード
            │  (GitHub OIDC トークンで認証 → シークレット設定不要)
            ▼
prperf サーバー
  ├─ base と PR を比較
  └─ Check Run / PR コメントに結果を通知
```

## 使う側の体験

1. GitHub App をリポジトリにインストール
2. 提供する GitHub Action をワークフローに数行追加
3. PR を作ると Check と PR コメントに結果が付く

## なぜ信頼できるか(設計の勘所)

- **回帰の判定は、決定的な指標を軸に。** rperf が測る中心は時間(フレーム
  グラフ)ですが、CI の実行時間は ±10〜20% ブレます。そこで**判定では**、
  ブレない **アロケーション数・GC 回数**を重視します(時間は参考)。結果には
  時間・GC・アロケーションのすべてと、フレームグラフが出ます。
- **シークレット不要。** 認証は GitHub Actions の OIDC トークン。API キーの
  発行・管理が要りません。
- **静かな通知。** Check Run は常設の置き場(通知ゼロ)、コメントは閾値超過時
  のみ・PR につき 1 通を編集。
- **「なぜ」まで分かる。** フレームグラフ diff で、重くなったメソッドを特定
  できます。

## 誰のためか

- **public な gem / ライブラリの作者** — 無料。性能回帰を PR で止めたい OSS。
- **private アプリで性能を気にするチーム** — 性能が UX や売上に直結する
  Rails アプリなど(有料プラン。現在は無料βのため public 専用)。

## このマニュアルの読み進め方

- **登録編** — インストールとワークフロー追加
- **ベンチマークの書き方** / **各クイックスタート(Rails ほか)** — 何を測るか
- **読み方編** / **フレームグラフの読み方** — 結果の読み取り

次は「登録編」へ。
